# Phase 1 Loop 1 — 验收标准（Acceptance Criteria）

版本：1.0.0 | 最后更新：2026-05-07

格式：Given / When / Then。标记 [P0] 为上线前必须通过的项目。

---

## 模块一：消费者浏览表盘列表

### AC-C01：消费者能看到表盘列表 [P0]

**Given** 数据库中已存在至少一条已上架的表盘记录  
**When** 任意用户（未登录）访问 `/`  
**Then**
- 页面在 3 秒内渲染出至少一张 `WatchfaceCard`
- 每张卡片展示：封面图、表盘标题、价格（0 分显示"免费"，其余显示格式化金额）
- 封面图宽高比为 1:1（或接近圆形表盘比例），无拉伸变形
- 页面标题（`<title>`）包含站点名称

### AC-C02：消费者看到加载状态 [P0]

**Given** 用户访问 `/`  
**When** API 请求发出但尚未返回时  
**Then**
- 页面显示骨架屏（`WatchfaceGridSkeleton`），卡片位置与真实卡片一致
- 不显示空态插图或错误提示

### AC-C03：表盘列表空态 [P0]

**Given** 数据库中没有任何已上架的表盘  
**When** 用户访问 `/`  
**Then**
- 显示 `EmptyState` 组件，包含"暂无表盘"主文案
- 不显示骨架屏、不报错
- 页面布局不破碎

### AC-C04：列表页网络错误处理 [P0]

**Given** API 服务不可用（网络超时或 5xx）  
**When** 用户访问 `/`  
**Then**
- 显示 `ErrorState` 组件，包含错误提示文案
- 页面上有可点击的"重试"按钮
- 点击"重试"后重新发起 API 请求

### AC-C05：无需登录访问 [P0]

**Given** 用户未登录（无 Session / Token）  
**When** 用户访问 `/`  
**Then**
- 页面正常渲染，不跳转到登录页
- 不显示任何"请登录"的提示或遮罩

## 模块二：设计师认证（注册 / 登录）

### AC-A01：设计师能成功注册 [P0]

**Given** 用户访问 `/auth/register` 并填写有效邮箱和密码（≥8 字符）  
**When** 点击"注册"按钮  
**Then**
- 系统创建新账户
- 用户自动登录（无需再手动登录）
- 页面跳转至注册前的目标页（如 `/designer/upload`）或首页

### AC-A02：注册邮箱已存在 [P0]

**Given** 用户填写了已注册的邮箱  
**When** 点击"注册"按钮  
**Then**
- 表单下方显示行内错误："该邮箱已被注册，请直接登录"
- 页面不跳转，表单数据保留

### AC-A03：设计师能成功登录 [P0]

**Given** 用户访问 `/auth/login` 并填写已注册的邮箱和正确密码  
**When** 点击"登录"按钮  
**Then**
- 用户登录成功
- 页面跳转至登录前的目标页（如 `/designer/upload`）或首页
- Header 中显示用户已登录状态

### AC-A04：登录密码错误 [P0]

**Given** 用户填写了已注册邮箱但密码错误  
**When** 点击"登录"按钮  
**Then**
- 表单下方显示行内错误："邮箱或密码不正确"（不区分具体哪个错，防止用户枚举）
- 页面不跳转，邮箱字段数据保留，密码字段清空

### AC-A05：表单字段校验 [P0]

**Given** 用户提交登录或注册表单时邮箱格式非法（如 "abc"）或密码为空  
**When** 点击提交按钮  
**Then**
- 前端即时（提交时）显示字段级错误提示
- 不发起 API 请求
- 页面自动滚动至第一个错误字段

## 模块三：设计师上传表盘

### AC-U01：未登录访问上传页被重定向 [P0]

**Given** 用户未登录  
**When** 用户访问 `/designer/upload`（直接输入 URL 或点击链接）  
**Then**
- 页面立即跳转至 `/auth/login?redirect=/designer/upload`
- 登录成功后自动跳转回 `/designer/upload`
- 不展示任何上传表单内容

### AC-U02：上传成功并自动上架 [P0]

**Given** 已登录设计师填写了有效的上传表单（标题非空、.prg 文件 ≤10MB、封面图 ≤2MB 且为 jpg/png）  
**When** 点击"提交"按钮，上传完成  
**Then**
- 页面展示成功反馈（`UploadSuccessCard` 或成功 Toast）
- 成功提示中包含"表盘已上架"类文案
- 表盘立即出现在消费者首页（`/`）的列表中，无需人工审核
- 提供"继续上传"和"查看表盘"两个后续操作入口

### AC-U03：上传后消费者立即可见 [P0]

**Given** 设计师刚刚完成一次成功上传  
**When** 消费者（可以是同一浏览器的无痕窗口）访问 `/`  
**Then**
- 刚上传的表盘出现在列表中
- 封面图显示正常（非破图）
- 标题与上传时填写的一致

### AC-U04：.prg 文件超出大小限制 [P0]

**Given** 设计师选择了一个大小超过 10MB 的 .prg 文件  
**When** 选择文件时（前端即时校验）或点击提交后（API 校验）  
**Then**
- 显示错误提示："文件大小超出 10MB 限制，请压缩后重新上传"
- 前端校验：错误显示在 `PrgFileField` 下方，不发起 API 请求
- API 校验（前端未拦截时）：显示 Toast，variant 为 destructive
- 表单其余数据不丢失

### AC-U05：.prg 文件格式错误 [P0]

**Given** 设计师选择了非 .prg 后缀的文件（如 .zip、.exe）  
**When** 选择文件时（前端即时校验）  
**Then**
- 显示错误提示："仅支持 .prg 格式文件"
- 不允许该文件被选入表单
- 或通过 `accept=".prg"` 限制系统文件选择对话框（但不完全依赖此约束）

### AC-U06：封面图格式错误 [P0]

**Given** 设计师选择了非 jpg/png 格式的封面图（如 .gif、.webp）  
**When** 选择文件时（前端即时校验）  
**Then**
- 显示错误提示："封面图仅支持 JPG 或 PNG 格式"
- 文件不被接受，不显示预览

### AC-U07：封面图超出大小限制 [P0]

**Given** 设计师选择了大小超过 2MB 的封面图  
**When** 选择文件时（前端即时校验）  
**Then**
- 显示错误提示："封面图大小超出 2MB 限制"
- 文件不被接受，不显示预览

### AC-U08：必填字段为空时阻止提交 [P0]

**Given** 设计师未填写标题，或未选择 .prg 文件，或未选择封面图  
**When** 点击"提交"按钮  
**Then**
- 不发起 API 请求
- 对应字段下方显示必填错误提示
- 页面滚动至第一个未填字段

### AC-U09：上传进度展示 [P1]

**Given** 设计师提交了有效表单且文件正在上传  
**When** 文件传输进行中  
**Then**
- 显示 `UploadProgressBar`，进度从 0% 变化至 100%
- 提交按钮处于禁用状态（`disabled`），防止重复提交
- 按钮文字变为"上传中…"

### AC-U10：服务器未知错误 [P1]

**Given** API 返回 500 或网络超时  
**When** 设计师提交表单后  
**Then**
- 显示 Toast："上传失败，请稍后重试"，variant 为 destructive
- 表单数据保留，设计师无需重新填写
- 提交按钮恢复为可点击状态

### AC-U11：价格字段校验 [P1]

**Given** 设计师在价格字段输入非数字字符或负数  
**When** 提交表单时  
**Then**
- 显示字段级错误："请输入有效价格（单位：美分，最小值为 0）"
- 不发起 API 请求
