# Phase 1 Loop 1 — 用户流程

版本：1.0.0 | 最后更新：2026-05-07

---

## 一、消费者流：浏览表盘列表

目标：任何人无需登录，访问首页，浏览表盘列表，点击查看详情（详情页为占位）。

```
[用户打开浏览器]
        │
        ▼
  ① 首页 / 列表页
  URL: /
  页面: WatchfaceListPage
  动作: 自动加载表盘列表
        │
        ├── 加载中 → 显示骨架屏（SkeletonCard × N）
        │
        ├── 无数据 → 显示空态插图 + 文案"暂无表盘，等待设计师上传"
        │
        ├── 加载失败 → 显示错误提示 + 重试按钮
        │
        └── 加载成功 → 渲染表盘卡片列表（WatchfaceCard）
                │
                ▼
          ② 点击表盘卡片
          URL: /watchfaces/:id
          页面: WatchfaceDetailPage（占位）
          动作: 展示"详情页建设中"占位内容
```

### 消费者流步骤表

| 步骤 | URL 路径 | 页面名称 | 主要动作 | 备注 |
|------|----------|----------|----------|------|
| 1 | `/` | WatchfaceListPage | 加载并展示表盘卡片列表 | 无需登录；移动端优先 |
| 2 | `/watchfaces/:id` | WatchfaceDetailPage | 占位展示 | Phase 1 仅占位，后续版本实现 |

## 二、设计师流：注册/登录并上传表盘

目标：设计师注册账号（或已有账号则直接登录），填写上传表单，提交后表盘自动上架，消费者立即可见。

```
[设计师打开浏览器]
        │
        ▼
  ① 上传页入口
  URL: /designer/upload
  动作: 检查登录状态
        │
        ├── 未登录 ──────────────────────────────────────────────────────┐
        │                                                               │
        ▼                                                               │
  ② 登录/注册页                                                         │
  URL: /auth/login  或  /auth/register                                  │
  页面: AuthPage（登录/注册 Tab 切换）                                   │
  动作:                                                                 │
    - 填写邮箱 + 密码                                                    │
    - 点击"登录"或"注册"                                                │
        │                                                               │
        ├── 认证失败 → 行内错误提示（邮箱未注册 / 密码错误）             │
        │                                                               │
        └── 认证成功 ──────────────────────────────────────────────────┘
                │
                ▼（redirect 回 /designer/upload）
  ③ 上传页
  URL: /designer/upload
  页面: UploadPage
  动作: 填写上传表单
        │
        ├── 标题（必填）
        ├── 描述（选填）
        ├── .prg 文件（必填，≤10MB）
        ├── 封面图（必填，≤2MB，jpg/png）
        ├── 目标设备（多选，选填）
        └── 价格（数字，单位：美分；0 = 免费）
                │
                ▼
  ④ 点击"提交"按钮
  动作: 前端校验 → 提交到 API
        │
        ├── 前端校验失败 → 字段级行内错误，滚动至第一个错误字段
        │
        ├── 上传中 → 提交按钮变为加载状态；显示进度条（文件上传）
        │
        ├── API 返回错误
        │       ├── 文件过大（>10MB） → Toast: "文件超出 10MB 限制"
        │       ├── 格式错误（非 .prg）→ Toast: "仅支持 .prg 格式"
        │       ├── 图片格式错误 → Toast: "封面图仅支持 JPG/PNG"
        │       └── 其他服务器错误 → Toast: "上传失败，请稍后重试"
        │
        └── 上传成功
                │
                ▼
  ⑤ 成功反馈页 / 成功 Toast
  URL: /designer/upload/success  或 Toast + 重定向至 /
  页面: UploadSuccessPage（或内联成功状态）
  动作:
    - 显示成功提示："表盘已上架，消费者现在即可看到"
    - 提供两个操作按钮：
        a. "继续上传"→ 重置表单 / 返回 /designer/upload
        b. "查看表盘"→ 跳转至首页 /
```

### 设计师流步骤表

| 步骤 | URL 路径 | 页面名称 | 主要动作 | 备注 |
|------|----------|----------|----------|------|
| 1 | `/designer/upload` | UploadPage | 检查登录态；未登录则重定向 | 服务端或客户端鉴权均可 |
| 2 | `/auth/login` | AuthPage | 邮箱+密码登录 | Tab 切换到注册 |
| 2a | `/auth/register` | AuthPage | 邮箱+密码注册 | 注册成功后自动登录 |
| 3 | `/designer/upload` | UploadPage | 填写上传表单 | redirect 回此页 |
| 4 | `/designer/upload` | UploadPage | 提交表单、展示上传进度 | 前端校验 + API 调用 |
| 5 | `/designer/upload/success` 或 Toast | UploadSuccessPage | 展示成功反馈；提供后续操作 | 自动上架，无审核 |
